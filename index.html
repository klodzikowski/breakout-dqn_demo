🧠 What is a DQN?

A Deep Q-Network is a neural network that learns to play games by trial and error. Rather than being told the rules, the agent repeatedly plays, observes what happens, and gradually figures out which actions lead to the highest long-term score. It’s the same family of algorithms DeepMind used to master Atari games in 2015.

In classic Q-learning you’d store a table of “how good is each action in each state?”. That works for small problems, but Breakout has far too many possible states. A neural network replaces the table — it takes in a description of the current game state and outputs a Q-value for each possible action (left, stay, right). The agent then picks the action with the highest predicted value.

⚙️ How Training Works

Each training step follows a cycle:

  1. Observe — The agent reads 10 features from the game: paddle position, ball position and velocity, a predicted landing position (where the ball will cross the paddle line, accounting for wall bounces), and the distribution of remaining bricks.
  2. Choose an action — Early on, the agent mostly picks random actions (exploration). As epsilon decays, it increasingly trusts its own Q-value predictions (exploitation).
  3. Collect the reward — Clearing a brick is worth +3, catching the ball +1, losing the ball −5. Small shaping bonuses reward moving towards where the ball will land, giving the network a clearer learning signal.
  4. Store the experience — The (state, action, reward, next state) tuple is saved to a replay buffer. During training, random batches are drawn from this buffer — this breaks the correlation between consecutive frames and stabilises learning.
  5. Update the network — Using the Bellman equation Q = r + γ · max Q(s′), the network’s weights are adjusted so its predictions better match the actual returns. A separate target network (a frozen copy, refreshed periodically) provides stable targets to train against.

🎯 Tips for Experimenting

Hover over any hyperparameter slider above for a detailed explanation. Here are some experiments worth trying:

  1. Speed up exploration — Lower the epsilon decay (e.g. 0.990) so the agent commits to its learnt policy sooner. It will learn faster, but may miss strategies it hasn’t discovered yet.
  2. Slow down learning — Drop the learning rate to 0.001. Training takes longer but the agent is less likely to “unlearn” good behaviours it already found.
  3. Make it forgetful — Shrink the replay buffer to 2,000. The agent trains only on very recent experiences, which can speed up adaptation but makes training more volatile.
  4. Destabilise on purpose — Set the target update interval very low (50 steps). You’ll see Q-values start oscillating — this is exactly why DQN needs a stable target network.
Reset and compare: After each experiment, hit Reset and train with the default settings side by side. Comparing the training curves is the best way to build intuition for what each hyperparameter does.
