🧠 How Q-Learning Works (click to expand) (click to collapse)

Q-Table: A table storing how "good" each action is from each state. Starts at zero (knows nothing).

Exploration vs Exploitation: Early on, the agent picks random actions (exploration). Over time, it increasingly picks the best known action (exploitation).

Update Rule: Q(s,a) = (1-α)Q(s,a) + α(r + γ max Q(s',a'))

Policy Arrows: The colors on the grid show the best action per state based on current Q-values.

Goal: Navigate from S (top-left) to G (bottom-right) without falling into H (holes).

💡 Tips: Try changing one parameter at a time and observe how it affects learning!
